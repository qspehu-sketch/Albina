# Frontend Design

Create frontend UI with a strong point of view, not generic filler. Deliver working code that is visually cohesive, usable, and adapted to the product context.

## When To Use

Apply this skill when the request involves:
- building a web page, component, section, catalog, or landing page
- redesigning an existing interface
- improving visual quality, spacing, typography, or atmosphere
- adapting a UI to a specific brand, mood, or audience
- making a frontend feel less templated or less AI-generated

## Quick Workflow

1. Identify the product context.
2. Choose one clear aesthetic direction.
3. Translate that direction into layout, typography, color, and motion decisions.
4. Implement production-ready code that matches the stack already in the project.
5. Check responsiveness, contrast, and visual consistency before finishing.

## Design Thinking

Before coding, define these four things:
- **Purpose**: what the interface helps the user do
- **Audience**: who will use it and what level of trust or emotion it should create
- **Constraints**: stack, performance, accessibility, assets, and responsive needs
- **Signature**: the one memorable visual trait that makes the design specific

Choose a deliberate direction and commit to it. Examples:
- soft botanical
- editorial luxury
- playful handcrafted
- industrial utility
- retro-futurist
- brutal minimal
- art deco geometry

Bold and quiet styles can both work. The requirement is intentionality.

## Implementation Rules

- Match the existing stack first; do not force a framework migration unless the user asks.
- Prefer reusable classes, tokens, and components over one-off styling.
- Use CSS variables or theme tokens for repeated colors, spacing, and shadows.
- Keep interactions lightweight and purposeful.
- Make the layout responsive by default.
- Preserve readability and accessibility even in expressive designs.

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Use type choices with character. Pair display and body styles intentionally. Avoid default-looking combinations unless the product truly needs neutrality.
- **Color**: Build around a dominant palette with clear accents. Favor coherent systems over many unrelated colors.
- **Composition**: Use spacing, rhythm, and hierarchy deliberately. A simple layout still needs tension and structure.
- **Backgrounds**: Add atmosphere where useful: gradients, textures, glow, pattern, blur, or depth layers that support the concept.
- **Motion**: Use animation to reinforce hierarchy and feedback, not to decorate every element.
- **Details**: Borders, radii, shadows, hover states, and empty states should feel part of the same system.

## Anti-Patterns

Avoid:
- generic hero sections with stock SaaS styling
- arbitrary gradients with no connection to the brand
- overcrowding every section with effects
- mixing too many visual languages in one screen
- default fonts and spacing that make the UI feel unfinished
- visual polish that hurts readability or mobile behavior

## Delivery Checklist

Before finishing, verify:
- the design has one clear aesthetic direction
- the main action is visually obvious
- spacing and type scale feel consistent
- the page works on mobile and desktop
- styling choices are reusable, not accidental
- the final UI feels specific to the user request

## Output Expectations

When applying this skill:
- briefly name the chosen visual direction
- explain the key design choices in one or two sentences if helpful
- implement the result directly in code
- mention any asset or dependency gaps only when they actually block quality
